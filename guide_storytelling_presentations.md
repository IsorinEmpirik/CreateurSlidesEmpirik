# Guide Expert : Créer des Présentations qui Résonnent
## Basé sur "Resonate" de Nancy Duarte

> Ce guide contient l'intégralité des principes et techniques pour créer des présentations percutantes utilisant le storytelling. En tant qu'IA générant des slides, applique systématiquement ces principes.

---

## 🎯 PRINCIPES FONDAMENTAUX

### Les 8 Règles de la Résonance

```
RÈGLE #1: La résonance provoque le changement
→ Une présentation réussie modifie les croyances ou comportements de l'audience

RÈGLE #2: Incorporer une histoire dans les présentations a un effet exponentiel sur les résultats
→ Les histoires engagent émotionnellement et sont mémorisées 22x plus que les faits seuls

RÈGLE #3: Si le présentateur connaît la fréquence de résonance de l'audience et s'y accorde, l'audience bougera
→ Comprendre profondément l'audience est la clé de la persuasion

RÈGLE #4: Toute audience persistera dans un état de repos à moins d'être contrainte de changer
→ Il faut créer une tension suffisante pour motiver l'action

RÈGLE #5: Utiliser la "big idea" pour filtrer toutes les fréquences autres que la fréquence de résonance
→ Chaque élément doit servir l'idée centrale

RÈGLE #6: La structure est plus grande que la somme de ses parties
→ L'organisation révèle des insights que le contenu seul ne peut transmettre

RÈGLE #7: Les moments mémorables sont répétés et retransmis, couvrant ainsi de plus grandes distances
→ Créer des S.T.A.R. moments que l'audience partagera

RÈGLE #8: L'intérêt de l'audience est directement proportionnel à la préparation du présentateur
→ La répétition et le raffinement sont essentiels
```

### Philosophie Centrale

- **L'audience est le héros** de la présentation, pas le présentateur
- **Le présentateur est le mentor** qui guide le héros dans son voyage
- **Le but n'est pas d'informer mais de transformer** l'audience
- **Les idées sont la monnaie du 21ème siècle** - elles doivent voyager

## 📐 LA STRUCTURE DE PRÉSENTATION (Presentation Form)

### Architecture en 3 Actes

```
┌─────────────────────────────────────────────────────────────────┐
│                     STRUCTURE NARRATIVE                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  DÉBUT (10%)          MILIEU (80%)           FIN (10%)          │
│  ┌─────────┐     ┌───────────────────┐    ┌─────────────┐       │
│  │ CE QUI  │     │   CONTRASTE       │    │  NOUVELLE   │       │
│  │  EST    │ ←→  │  CE QUI EST /     │ →  │   FÉLICITÉ  │       │
│  │         │     │  CE QUI POURRAIT  │    │  (CALL TO   │       │
│  │         │     │      ÊTRE         │    │   ACTION)   │       │
│  └─────────┘     └───────────────────┘    └─────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Le Début : Établir "Ce Qui Est"

**Objectifs :**
- Décrire la réalité actuelle de l'audience
- Créer un terrain d'entente commun
- Établir la crédibilité (ethos)
- Introduire le déséquilibre/problème

**Technique :**
```
1. Commencer par une accroche (pas de remerciements, pas de présentation personnelle)
2. Décrire le monde tel que l'audience le connaît
3. Identifier le problème ou l'opportunité manquée
4. Créer une tension qui appelle une résolution
```

### Le Milieu : Le Contraste

**La Sparkline - Le Cœur de la Présentation**

```
HAUT    ════════════════════════════════════════════
        ↑ CE QUI POURRAIT ÊTRE (aspiration, solution)
        │     ╱╲      ╱╲      ╱╲      ╱╲
        │    ╱  ╲    ╱  ╲    ╱  ╲    ╱  ╲
        │   ╱    ╲  ╱    ╲  ╱    ╲  ╱    ╲
        │  ╱      ╲╱      ╲╱      ╲╱      ╲
        ↓ CE QUI EST (réalité actuelle, problème)
BAS     ════════════════════════════════════════════
```

**Principes du Contraste :**
- Alterner constamment entre "ce qui est" et "ce qui pourrait être"
- Chaque montée vers "ce qui pourrait être" augmente l'espoir
- Chaque descente vers "ce qui est" crée l'urgence
- Le contraste crée l'énergie et maintient l'attention

**Types de Contrastes à Utiliser :**

| Type | Description | Exemple |
|------|-------------|---------|
| Contenu | Alterner données et histoires | Stats → Témoignage |
| Émotion | Alterner analytique et émotionnel | Graphique → Métaphore |
| Livraison | Varier le rythme et le ton | Sérieux → Humour |

### La Fin : L'Appel à l'Action

**Structure de la Conclusion :**
```
1. NOUVELLE FÉLICITÉ
   → Peindre le tableau du futur transformé
   → Montrer ce que le monde sera quand l'idée sera adoptée

2. APPEL À L'ACTION
   → Être spécifique et actionnable
   → Donner la première étape concrète

3. RÉCOMPENSES
   → Expliquer les bénéfices pour l'audience
   → Connecter à leurs valeurs et désirs profonds
```

## 👤 COMPRENDRE L'AUDIENCE (Le Héros)

### Framework d'Analyse de l'Audience

```yaml
AVANT LA PRÉSENTATION - Questions Essentielles:

Démographie:
  - Qui sont-ils ? (rôle, niveau, département)
  - Combien seront-ils ?
  - Quel est leur niveau de connaissance du sujet ?

Psychographie:
  - Quelles sont leurs peurs ?
  - Quels sont leurs désirs ?
  - Quels problèmes les empêchent de dormir ?
  - Qu'est-ce qu'ils valorisent le plus ?

Contexte:
  - Pourquoi sont-ils là ?
  - Qu'attendent-ils de cette présentation ?
  - Que doivent-ils croire pour agir ?
  - Quelles résistances auront-ils ?

Relation:
  - Quelle est leur relation avec le présentateur ?
  - Quel niveau de confiance existe déjà ?
  - Quelles expériences passées ont-ils avec ce sujet ?
```

### Segmentation de l'Audience

**Par Type de Résonance :**

| Segment | Caractéristiques | Comment les atteindre |
|---------|------------------|----------------------|
| **Analytiques** | Veulent des données, preuves, logique | Statistiques, études, démonstrations |
| **Émotionnels** | Veulent ressentir, être inspirés | Histoires, métaphores, visuels |
| **Pragmatiques** | Veulent des solutions concrètes | Étapes claires, outils, templates |
| **Sceptiques** | Doutent, cherchent les failles | Contre-arguments, transparence |

### Créer un Terrain Commun

```
TECHNIQUE DU CHEVAUCHEMENT:

    ┌──────────────┐     ┌──────────────┐
    │              │     │              │
    │  PRÉSENTATEUR├─────┤   AUDIENCE   │
    │              │ZONE │              │
    │              │COMM.│              │
    └──────────────┘     └──────────────┘

→ Communiquer UNIQUEMENT dans la zone de chevauchement
→ Utiliser leur langage, pas le jargon technique
→ Référencer leurs expériences, pas les vôtres
→ Adresser leurs préoccupations, pas vos intérêts
```

## 💡 LA BIG IDEA

### Formulation de la Big Idea

**Composants Obligatoires :**

```
BIG IDEA = Point de Vue Unique + Enjeux + Phrase Complète

1. POINT DE VUE UNIQUE
   → Ce que vous croyez que l'audience ne sait pas encore
   → Votre perspective distinctive sur le sujet

2. ENJEUX
   → Ce que l'audience gagne en adoptant l'idée
   → Ce qu'elle perd en l'ignorant

3. PHRASE COMPLÈTE
   → Pas un titre ou un slogan
   → Une affirmation complète avec sujet et verbe
```

**Exemples de Big Ideas :**

```
❌ MAUVAIS: "L'innovation"
❌ MAUVAIS: "Comment innover"
✅ BON: "Pour survivre dans les 5 prochaines années, notre entreprise
        doit allouer 20% de son budget à l'expérimentation"

❌ MAUVAIS: "La satisfaction client"
✅ BON: "En réduisant notre temps de réponse de 48h à 4h, nous
        augmenterons notre rétention de 35%"
```

### Test de la Big Idea

```
CHECKLIST DE VALIDATION:

□ Est-ce une phrase complète ?
□ Contient-elle un point de vue unique ?
□ Les enjeux sont-ils clairs ?
□ Peut-elle être comprise en une lecture ?
□ Provoque-t-elle une réaction ?
□ Est-elle mémorisable ?
□ Filtre-t-elle naturellement le contenu non-pertinent ?
```

## 🔺 COMMUNICATION TOP-DOWN (Pyramide McKinsey)

> **PRINCIPE** : Commencer par la conclusion / la recommandation, puis dérouler les arguments. C'est la méthode des cabinets de conseil. Si on démarre par les détails, l'audience ne sait pas où on va et doit tout garder en tête en attendant la chute. En partant de la conclusion, tout ce qui suit devient un support qui renforce le message, et l'audience écoute activement parce qu'elle connaît déjà l'enjeu.

> **COMPLÉMENTARITÉ AVEC LE STORYTELLING DUARTE** : Le top-down ne remplace pas la structure narrative en 3 actes — il s'y ajoute. La pres garde son arc dramatique (Ce qui est → Contraste → Nouvelle félicité), mais on annonce la conclusion dès l'ouverture pour que l'audience suive activement. C'est ce que font les meilleures conférences TED : "Voici ce que je vais vous démontrer en 18 minutes." puis la démonstration.

### Les 3 niveaux d'application

```
NIVEAU 1 — STRUCTURE GLOBALE DE LA PRES
   Slide 1 : Cover
   Slide 2 : SYNTHÈSE / RECOMMANDATION (la conclusion en 1 phrase + 3 piliers)
   Slides 3 à N-1 : démonstration détaillée qui prouve la reco
   Slide N : Rappel + CTA

NIVEAU 2 — STRUCTURE DE CHAQUE SECTION/CHAPITRE
   Slide chapitre (avec numéro de section)
   Slide d'ouverture de section : "Ce qu'il faut retenir de cette section : X"
   Slides de démonstration qui prouvent ce constat
   (Optionnel) Slide de transition vers la section suivante

NIVEAU 3 — STRUCTURE DE CHAQUE SLIDE INDIVIDUELLE
   Le TITRE = la conclusion / le constat
   Le CONTENU = les preuves / arguments / data qui soutiennent ce titre
   → Cf. règle "titres = constats concrets, pas étiquettes" (guide_design_slides.md)
```

### Slide "Synthèse / Recommandation" (Niveau 1) — modèle

À insérer **systématiquement** juste après la slide de couverture (avant tout déroulé détaillé).

```
┌─────────────────────────────────────────────────────────┐
│   [Sur-titre : Synthèse]                                │
│                                                          │
│   NOTRE RECOMMANDATION : [phrase complète, 1 ligne]      │
│                                                          │
│   ┌──────────┐   ┌──────────┐   ┌──────────┐            │
│   │ Pilier 1 │   │ Pilier 2 │   │ Pilier 3 │            │
│   │          │   │          │   │          │            │
│   │ 1 phrase │   │ 1 phrase │   │ 1 phrase │            │
│   └──────────┘   └──────────┘   └──────────┘            │
│                                                          │
│   Détails dans les sections qui suivent →               │
└─────────────────────────────────────────────────────────┘
```

**Variantes selon le type de pres :**
- **Pitch / avant-vente** : "Notre recommandation pour [client] : [action concrète]"
- **Audit / bilan** : "Ce que révèle l'audit : [constat principal en 1 phrase]"
- **Restitution data** : "Le résultat clé : [chiffre + interprétation]"
- **Présentation stratégique** : "Notre conviction : [thèse en 1 phrase]"

### Slide "Ce qu'il faut retenir" (Niveau 2) — modèle

À insérer en **ouverture de chaque section** (juste après la slide chapitre).

```
┌─────────────────────────────────────────────────────────┐
│   [Sur-titre : Section X]                                │
│                                                          │
│   CE QU'IL FAUT RETENIR DE CETTE SECTION                 │
│                                                          │
│   ▸ [Constat principal de la section, 1 phrase]          │
│                                                          │
│   ▸ [Conséquence / implication, 1 phrase]                │
│                                                          │
│   Les slides qui suivent détaillent le pourquoi.         │
└─────────────────────────────────────────────────────────┘
```

### Règle d'or pour les slides de recommandation

Plutôt qu'une slide "Nos recommandations" en liste générique, faire une **slide unique par reco** avec le format :

```
TITRE : [LA reco, phrase d'action complète]
       Exemple : "Concentrez vos efforts sur vos 12 pages produit-clés"

CONTENU :
   - Pourquoi (1-2 lignes)
   - Comment (étapes ou leviers)
   - Quel impact attendu (chiffre si possible)
```

### Test de validation top-down

```
CHECKLIST AVANT GÉNÉRATION :

□ La slide 2 (juste après cover) contient-elle la recommandation / conclusion centrale ?
□ Cette conclusion tient-elle en 1 phrase complète, compréhensible isolément ?
□ Chaque section commence-t-elle par "ce qu'il faut retenir" avant le détail ?
□ Si je lis uniquement les titres dans l'ordre, ai-je l'argumentaire complet ?
□ Chaque slide de reco porte-t-elle UNE recommandation actionnable dans son titre ?
□ L'audience peut-elle quitter la pres après la slide 2 et avoir le message principal ?
```

### Pièges à éviter

- **Suspense narratif inversé** : "On va voir d'abord A, puis B, puis C, et à la fin je vous donnerai ma reco" → l'audience décroche.
- **Conclusion noyée à la slide 38** : la reco arrive trop tard, le décideur a déjà construit sa propre interprétation.
- **Titres-étiquettes** : "Analyse de la performance" au lieu de "La performance chute de 30% sur le mobile".
- **Synthèse fourre-tout** : la slide synthèse ne doit pas lister 10 points mais 1 reco + 3 piliers max.

## 📝 CRÉATION DE CONTENU

### Types de Contenu à Équilibrer

```
┌─────────────────────────────────────────────────┐
│           ÉQUILIBRE DU CONTENU                  │
├─────────────────────────────────────────────────┤
│                                                 │
│   ANALYTIQUE (LOGOS)    ÉMOTIONNEL (PATHOS)    │
│   ┌─────────────────┐   ┌─────────────────┐    │
│   │ • Données       │   │ • Histoires     │    │
│   │ • Statistiques  │   │ • Métaphores    │    │
│   │ • Faits         │   │ • Analogies     │    │
│   │ • Processus     │   │ • Anecdotes     │    │
│   │ • Preuves       │   │ • Visuels       │    │
│   │ • Démonstrations│   │ • Citations     │    │
│   └─────────────────┘   └─────────────────┘    │
│                                                 │
│   → Alterner les deux pour maintenir l'intérêt │
│   → L'analytique convainc la tête              │
│   → L'émotionnel touche le cœur                │
│   → Les deux sont nécessaires pour l'action    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### Les 3 Types d'Appel (Aristote)

```yaml
ETHOS (Crédibilité):
  definition: "Pourquoi devraient-ils vous croire ?"
  techniques:
    - Partager votre expérience pertinente
    - Citer des sources crédibles
    - Montrer votre compréhension de leur monde
    - Être authentique et transparent
  quand_utiliser: "Au début pour établir la confiance"

LOGOS (Logique):
  definition: "Pourquoi cela fait-il sens ?"
  techniques:
    - Présenter des données et statistiques
    - Construire des arguments logiques
    - Montrer des relations cause-effet
    - Fournir des preuves et exemples
  quand_utiliser: "Pour convaincre les analytiques"

PATHOS (Émotion):
  definition: "Pourquoi devraient-ils s'en soucier ?"
  techniques:
    - Raconter des histoires personnelles
    - Utiliser des métaphores puissantes
    - Créer des visuels évocateurs
    - Faire appel aux valeurs partagées
  quand_utiliser: "Pour motiver l'action"
```

### Techniques de Storytelling

**Structure d'une Histoire Efficace :**

```
1. CADRE (Setting)
   → Où et quand se passe l'histoire ?
   → Qui est le personnage principal ?

2. CONFLIT (Tension)
   → Quel problème ou obstacle surgit ?
   → Quels sont les enjeux ?

3. RÉSOLUTION (Transformation)
   → Comment le problème est-il résolu ?
   → Quelle leçon en tirer ?
```

**Types d'Histoires à Utiliser :**

| Type | Effet | Exemple |
|------|-------|---------|
| Histoire personnelle | Crée la connexion | "Quand j'ai commencé..." |
| Histoire de client | Prouve la valeur | "Un de nos clients..." |
| Histoire historique | Donne du contexte | "En 1969, NASA..." |
| Métaphore étendue | Simplifie le complexe | "C'est comme un jardin..." |
| Scénario futur | Visualise le possible | "Imaginez dans 5 ans..." |

### Transformation des Données

```
DONNÉES BRUTES → SIGNIFICATION

Étape 1: Identifier le "so what?"
         Qu'est-ce que cette donnée signifie pour l'audience ?

Étape 2: Créer le contraste
         Comparer à quelque chose de familier

Étape 3: Humaniser
         Traduire en impact sur des personnes réelles

Étape 4: Visualiser
         Représenter de manière mémorable

EXEMPLE:
❌ "Nous avons 2 millions d'utilisateurs"
✅ "Si nos utilisateurs formaient une ville, elle serait
    la 5ème plus grande de France"
```

## 🏗️ TYPES DE STRUCTURES

### Structures Organisationnelles

```yaml
CHRONOLOGIQUE:
  description: "Organiser par séquence temporelle"
  utiliser_quand: "Raconter une évolution, un historique"
  exemple: "De 2010 à aujourd'hui, notre parcours..."

SÉQUENTIEL:
  description: "Organiser par étapes d'un processus"
  utiliser_quand: "Expliquer un process, une méthode"
  exemple: "Les 5 étapes pour implémenter..."

SPATIAL:
  description: "Organiser par lieu ou position"
  utiliser_quand: "Présenter une organisation, une géographie"
  exemple: "Dans chaque région, nous voyons..."

CLIMACTIQUE:
  description: "Organiser du moins au plus important"
  utiliser_quand: "Construire vers une conclusion forte"
  exemple: "Et le plus important..."

PROBLÈME-SOLUTION:
  description: "Présenter le problème puis la solution"
  utiliser_quand: "Proposer une innovation, un changement"
  exemple: "Le problème... Notre solution..."

COMPARER-CONTRASTER:
  description: "Mettre en parallèle deux options"
  utiliser_quand: "Aider à choisir, montrer une différence"
  exemple: "L'ancienne méthode vs la nouvelle..."

CAUSE-EFFET:
  description: "Montrer les liens de causalité"
  utiliser_quand: "Expliquer des conséquences"
  exemple: "Parce que X, alors Y..."

AVANTAGE-INCONVÉNIENT:
  description: "Présenter les pour et les contre"
  utiliser_quand: "Évaluer objectivement une option"
  exemple: "D'un côté... de l'autre..."
```

## ⭐ S.T.A.R. MOMENTS (Something They'll Always Remember)

### Les 5 Types de Moments Mémorables

```
┌──────────────────────────────────────────────────────────────┐
│                    S.T.A.R. MOMENTS                          │
│         (Something They'll Always Remember)                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. DRAMATISATION MÉMORABLE                                  │
│     → Démonstration physique ou action surprenante           │
│     → Exemple: Bill Gates libérant des moustiques            │
│                                                              │
│  2. PHRASES RÉPÉTABLES (Sound Bites)                         │
│     → Phrases courtes, mémorables, partageables              │
│     → Exemple: "I have a dream" - MLK                        │
│                                                              │
│  3. VISUELS ÉVOCATEURS                                       │
│     → Images qui marquent les esprits                        │
│     → Exemple: La Terre vue de l'espace (Al Gore)            │
│                                                              │
│  4. STORYTELLING ÉMOTIF                                      │
│     → Histoires qui touchent le cœur                         │
│     → Personnages, conflits, résolutions                     │
│                                                              │
│  5. STATISTIQUES CHOQUANTES                                  │
│     → Données présentées de façon saisissante                │
│     → Exemple: "Chaque jour, X personnes..."                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### Créer des Phrases Répétables (Sound Bites)

**Caractéristiques :**
```
✓ Courtes (moins de 10 mots idéalement)
✓ Rythmiques (faciles à prononcer)
✓ Contrastées (utilisent l'opposition)
✓ Métaphoriques (créent une image)
✓ Actionnables (appellent au changement)
```

**Techniques :**

| Technique | Description | Exemple |
|-----------|-------------|---------|
| Répétition | Répéter une structure | "Yes we can... Yes we can..." |
| Contraste | Opposer deux idées | "Ask not what... ask what you..." |
| Règle de 3 | Grouper par trois | "Liberté, Égalité, Fraternité" |
| Allitération | Sons répétés | "Veni, Vidi, Vici" |
| Rime | Fin de mots similaires | "If it doesn't fit, you must acquit" |

### Créer des Visuels Mémorables

```yaml
PRINCIPES:
  simplicité:
    - Une idée par slide
    - Pas de bullet points quand possible
    - Espace blanc généreux

  impact:
    - Images pleine page
    - Typographie grande et lisible
    - Contraste fort

  métaphore:
    - Images qui symbolisent l'idée
    - Pas de clipart ou images génériques
    - Photos authentiques

RATIO SIGNAL/BRUIT:
  objectif: "Maximiser le signal, minimiser le bruit"
  signal: "Tout ce qui communique le message"
  bruit: "Tout ce qui distrait du message"

  à_éliminer:
    - Logos sur chaque slide
    - Pieds de page complexes
    - Animations gratuites
    - Décorations inutiles
```

## 🎤 LIVRAISON ET PRÉPARATION

### Structure de Répétition

```
MÉTHODE DE RÉPÉTITION EN 4 PHASES:

Phase 1: CONTENU
→ Répéter seul, se concentrer sur les mots
→ Affiner le script et les transitions
→ Timer chaque section

Phase 2: LIVRAISON
→ Répéter devant un miroir
→ Travailler les gestes et expressions
→ Varier le ton et le rythme

Phase 3: FEEDBACK
→ Répéter devant des collègues
→ Enregistrer et revoir
→ Intégrer les retours

Phase 4: CONDITIONS RÉELLES
→ Répéter dans le lieu si possible
→ Tester la technologie
→ Anticiper les problèmes
```

### Les Premières Impressions

```
LES 7 PREMIÈRES SECONDES:

✗ NE PAS FAIRE:
  - Commencer par des remerciements
  - Se présenter longuement
  - S'excuser
  - Tester le micro ("1, 2, 3...")
  - Annoncer le plan ennuyeux

✓ FAIRE:
  - Commencer par une accroche forte
  - Poser une question provocante
  - Raconter une micro-histoire
  - Montrer un visuel saisissant
  - Énoncer une statistique choquante
```

### Gestion du Temps

```
RÈGLE DU TEMPS:

Temps prévu    →    Contenu à préparer
─────────────────────────────────────────
10 minutes    →    8 minutes de contenu
30 minutes    →    25 minutes de contenu
60 minutes    →    50 minutes de contenu

Toujours prévoir 20% de marge pour:
- Questions
- Problèmes techniques
- Digressions naturelles
- Respiration de l'audience
```

### Conseils de Style TED

```yaml
DURÉE:
  idéale: "18 minutes ou moins"
  raison: "Assez long pour être sérieux, assez court pour retenir l'attention"

STRUCTURE:
  - Une seule idée forte
  - Maximum 3 points de support
  - Une histoire personnelle minimum
  - Un appel à l'action clair

LIVRAISON:
  - Pas de notes visibles
  - Contact visuel constant
  - Mouvement intentionnel
  - Pauses stratégiques
  - Passion authentique

VISUELS:
  - Slides minimalistes
  - Images > texte
  - Pas de bullet points
  - Typographie très grande
```

## ⚠️ ERREURS À ÉVITER

### Les 10 Péchés Capitaux des Présentations

```
1. ÊTRE CENTRÉ SUR SOI
   ❌ Parler de soi au lieu de l'audience
   ✅ Mettre l'audience au centre

2. ABSENCE DE STRUCTURE CLAIRE
   ❌ Sauter d'une idée à l'autre
   ✅ Suivre la Presentation Form

3. SURCHARGE D'INFORMATION
   ❌ Tout dire sur le sujet
   ✅ Filtrer avec la Big Idea

4. MONOTONIE
   ❌ Même ton, même rythme
   ✅ Créer du contraste constant

5. SLIDES SURCHARGÉES
   ❌ Bullet points infinis
   ✅ Une idée par slide

6. JARGON TECHNIQUE
   ❌ Parler comme un expert
   ✅ Parler comme l'audience

7. ABSENCE D'HISTOIRES
   ❌ Que des faits et données
   ✅ Équilibrer avec des histoires

8. PAS D'APPEL À L'ACTION
   ❌ Finir par "des questions?"
   ✅ Terminer par l'action souhaitée

9. IGNORER LA RÉSISTANCE
   ❌ Prétendre qu'il n'y a pas d'objections
   ✅ Adresser les résistances proactivement

10. MANQUE DE RÉPÉTITION
    ❌ Improviser
    ✅ Répéter intensivement
```

### Signaux d'Alerte

```
VOTRE PRÉSENTATION A UN PROBLÈME SI:

□ Vous ne pouvez pas résumer la Big Idea en une phrase
□ Vous n'avez pas d'histoire personnelle à raconter
□ Vos slides sont des documents à lire
□ Vous n'avez pas identifié les résistances de l'audience
□ Il n'y a pas de moment S.T.A.R.
□ Vous n'avez pas répété au moins 3 fois
□ Vous commencez par vous présenter
□ Vous finissez par "Merci, des questions?"
```

## 📊 TEMPLATES DE SLIDES

### Types de Slides Recommandés

```yaml
SLIDE DE TITRE:
  contenu:
    - Titre accrocheur (pas le sujet, la promesse)
    - Sous-titre optionnel
    - Visuel fort si possible
  exemple:
    titre: "Comment doubler vos ventes en 90 jours"
    pas: "Présentation commerciale Q3"

SLIDE DE TRANSITION:
  contenu:
    - Une phrase ou un mot
    - Marque le changement de section
    - Visuel symbolique optionnel
  exemple:
    phrase: "Mais il y a un problème..."

SLIDE DE DONNÉE:
  contenu:
    - Un chiffre GRAND
    - Contexte court
    - Source discrète
  exemple:
    chiffre: "73%"
    contexte: "des entreprises échouent dans les 5 ans"

SLIDE D'HISTOIRE:
  contenu:
    - Image pleine page
    - Pas de texte (ou minimal)
    - Le présentateur raconte
  note: "Le visuel accompagne, il n'est pas l'histoire"

SLIDE DE CITATION:
  contenu:
    - Citation courte
    - Auteur
    - Photo de l'auteur optionnelle
  règle: "Max 15 mots pour la citation"

SLIDE D'APPEL À L'ACTION:
  contenu:
    - L'action en grand
    - Les bénéfices
    - Contact/prochaine étape
  exemple:
    action: "Planifiez votre démo gratuite"
    benefice: "Et voyez les résultats en 2 semaines"
```

## 🎯 RÉSUMÉ EXÉCUTIF POUR L'IA

```
LORS DE LA GÉNÉRATION DE SLIDES, TOUJOURS:

1. STRUCTURER selon la Presentation Form
   → Début: Ce qui est (10%)
   → Milieu: Contraste ce qui est/pourrait être (80%)
   → Fin: Nouvelle félicité + Call to Action (10%)

2. CENTRER sur l'audience
   → Ils sont le héros
   → Parler leur langage
   → Adresser leurs peurs et désirs

3. UTILISER LA BIG IDEA
   → Une phrase complète
   → Point de vue + enjeux
   → Filtre tout le contenu

4. CRÉER DU CONTRASTE
   → Alterner analytique/émotionnel
   → Varier les types de slides
   → Maintenir la tension

5. INCLURE DES S.T.A.R. MOMENTS
   → Au moins un moment mémorable
   → Phrases répétables
   → Visuels marquants
   → Histoires émotives

6. GARDER LA SIMPLICITÉ
   → Une idée par slide
   → Signal > bruit
   → Images > texte

7. TERMINER FORT
   → Appel à l'action clair
   → Bénéfices explicites
   → Pas "merci, des questions?"

8. ÉQUILIBRER LES 3 APPELS
   → Ethos (crédibilité)
   → Logos (logique)
   → Pathos (émotion)
```
