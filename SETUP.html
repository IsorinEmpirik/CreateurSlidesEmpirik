Setup — Créateur de Slides x Storytelling

Tout ce qu'il faut installer et configurer avant de générer ta première présentation.

Comment utiliser ce guide. On part du principe que tu as déjà Claude Code (extension VS Code) et que tu as ouvert ce dossier dedans. À partir de là, presque tout s'installe automatiquement via les prompts ci-dessous. Tu copies, tu colles dans Claude Code, et il vérifie + installe ce qui manque sans que tu aies à toucher au terminal.

1. Avant de commencer

Tu as reçu ce dossier sous forme de ZIP. Décompresse-le à un emplacement de ton choix (idéalement Desktop ou Documents) puis ouvre-le dans Claude Code (via VS Code). Une fois ouvert, Claude lit automatiquement CLAUDE.md qui contient toutes les règles Empirik — pas besoin de lui rappeler manuellement.

⚠ IMPORTANT

Merci de remplacer les clés API Gemini et OpenAI par les vôtres dès que possible

Les clés livrées dans clé-api-gemini.md et clé-api-openai.md sont celles personnelles de l'expéditeur. Si vous les utilisez en production, Yannick va voir une consommation grimper sur le compte de l'expéditeur alors que ce n'est pas lui qui consomme — ça va perturber le suivi des dépenses de l'agence.

  • clé-api-gemini.md et clé-api-openai.md servent à générer des images IA (notamment la slide de couverture et les visuels concepts). À remplacer par les vôtres rapidement :
    • Soit créer la vôtre : Gemini → aistudio.google.com/apikey · OpenAI → platform.openai.com/api-keys
    • Soit demander une clé personnelle à Yannick
  • clé-api-dataforseo.md : celle de Yannick directement, vous pouvez la garder telle quelle, pas de souci.

Une fois récupérée, éditer simplement le fichier clé-api-gemini.md et/ou clé-api-openai.md à la racine et y coller votre clé à la place. Détails techniques sur l'usage des clés en §3 étape 7.

2. Le prompt unique (le plus rapide)

Si tu veux que Claude Code fasse tout d'un coup — vérifier, installer ce qui manque, configurer et auditer — colle ce prompt. C'est la voie recommandée pour la majorité des utilisateurs.

Setup complet automatique
Setup environnement complet pour le projet Empirik de génération de slides. Étape par étape, vérifie et installe si manquant : (1) Node.js LTS ≥ 20 et npm (via winget sous Windows, brew sous macOS, package manager natif sous Linux), (2) Python 3.10+ et pip (idem), (3) LibreOffice (idem ; sous Windows confirme le chemin C:\Program Files\LibreOffice\program\soffice.exe), (4) packages npm globaux : pptxgenjs, react-icons, react, react-dom, sharp ; sous Windows configure aussi NODE_PATH=%APPDATA%\npm\node_modules, (5) packages Python pip : markitdown[pptx], Pillow, PyMuPDF, google-generativeai, matplotlib, (6) famille Poppins (ExtraLight 200, Light 300, Regular 400, Medium 500, SemiBold 600, Bold 700) — si manquante, télécharge depuis Google Fonts et installe (Windows : copie dans %LOCALAPPDATA%\Microsoft\Windows\Fonts qui ne demande pas admin ; macOS : ~/Library/Fonts), (7) validité des 3 clés API en lisant clé-api-gemini.md, clé-api-openai.md, clé-api-dataforseo.md et en faisant un ping minimal pour chacune. À la fin, génère un rapport checklist ✅/❌ avec actions correctives pour chaque ❌. Demande-moi confirmation avant toute installation qui pourrait demander mon mot de passe ou un accès admin.
Et si tu veux contrôler chaque étape ? Section §3 ci-dessous : un prompt par dépendance, à exécuter dans l'ordre. Utile si tu veux comprendre ce qui se passe ou si une étape échoue.

3. Setup étape par étape (option contrôle)

Pour chaque dépendance, un prompt dédié à coller dans Claude Code. À enchaîner dans l'ordre proposé. Claude vérifie l'état actuel avant toute installation.

1

Node.js + npm

Obligatoire

PptxGenJS (la librairie qui génère les .pptx) tourne sous Node.js. Sans Node, rien ne sort.

Prompt à coller dans Claude Code
Vérifie si Node.js (LTS ≥ 20) et npm sont installés sur cette machine. Si non, installe-les automatiquement (winget install OpenJS.NodeJS.LTS sous Windows ; brew install node sous macOS ; gestionnaire natif sous Linux). Demande-moi confirmation avant l'installation si elle nécessite un accès admin. Une fois fait, affiche les versions installées.
2

Python 3.10+ et pip

Obligatoire

Plusieurs scripts utilisent Python : génération d'images Gemini, extraction de texte des PPTX, conversion PDF → images pour la QA visuelle.

Prompt à coller dans Claude Code
Vérifie si Python ≥ 3.10 et pip sont installés et accessibles dans le PATH. Si manquant, installe-les automatiquement (winget install Python.Python.3.12 sous Windows en cochant "Add Python to PATH" ; brew install python sous macOS ; gestionnaire natif sous Linux). Demande-moi confirmation avant toute action admin. Affiche les versions à la fin.
3

LibreOffice

Obligatoire

Convertit les .pptx en PDF pour la QA visuelle (PDF → JPG via PyMuPDF). Sans LibreOffice, impossible de vérifier le rendu final des slides.

Prompt à coller dans Claude Code
Vérifie si LibreOffice est installé (cherche le binaire soffice/soffice.exe selon mon OS). Si non, installe-le automatiquement (winget install TheDocumentFoundation.LibreOffice sous Windows ; brew install --cask libreoffice sous macOS ; gestionnaire natif sous Linux). Si oui, affiche le chemin complet et teste avec une conversion .pptx → PDF minimale pour confirmer que ça fonctionne.
4

Dépendances npm globales

Obligatoire

PptxGenJS pour générer les .pptx, react-icons pour les icônes, sharp pour le traitement d'images. En global pour réutilisation sur tous tes projets.

Prompt à coller dans Claude Code
Installe les dépendances npm globales pour le projet Empirik : pptxgenjs, react-icons, react, react-dom, sharp. Sous Windows, configure aussi la variable d'environnement NODE_PATH vers %APPDATA%\npm\node_modules (commande : setx NODE_PATH "%APPDATA%\npm\node_modules"). Vérifie ensuite que pptxgenjs est bien trouvable en faisant un test require depuis Node.
5

Dépendances Python (pip)

Obligatoire

markitdown lit les .pptx (vérification du texte), PyMuPDF convertit PDF → JPG (QA visuelle slide par slide), google-generativeai génère les images Gemini (cover, sections, concepts), Pillow traite les images, matplotlib génère les graphes complexes en PNG (slope graph, line chart, dot plot, heatmap, waterfall — Voie 3 de génération, cf CLAUDE.md §6.6).

Prompt à coller dans Claude Code
Installe les dépendances Python pip pour le projet Empirik : markitdown[pptx], Pillow, PyMuPDF, google-generativeai, matplotlib. Si pip n'est pas reconnu, utilise python -m pip. Une fois installé, fais un test d'import de chaque package pour confirmer qu'ils marchent (attention : PyMuPDF s'importe sous le nom fitz).
6

Polices Poppins (toutes variantes)

Obligatoire

La charte Empirik utilise Poppins partout avec des variantes spécifiques (ExtraLight 200, Regular 400, Medium 500, SemiBold 600, Bold 700). Si elles ne sont pas installées, les .pptx tombent sur une police de substitution et le rendu est cassé (cf. erreur récurrente #2 du process anti-erreurs).

Prompt à coller dans Claude Code
Vérifie si la famille Poppins est installée sur ma machine, en particulier les variantes ExtraLight (200), Light (300), Regular (400), Medium (500), SemiBold (600), Bold (700) — toutes nécessaires pour la charte Empirik. Si certaines variantes manquent, télécharge la famille complète depuis Google Fonts (https://fonts.google.com/specimen/Poppins, ZIP "Download family"), dézippe, et installe les .ttf manquants : sous Windows, copie-les dans %LOCALAPPDATA%\Microsoft\Windows\Fonts (ne demande pas admin) ; sous macOS, copie-les dans ~/Library/Fonts ; sous Linux, dans ~/.local/share/fonts puis lance fc-cache -fv. Confirme à la fin que toutes les variantes sont reconnues par le système.
Si Claude n'arrive pas à installer les fonts automatiquement (cas rare sous Windows verrouillé) : télécharge la famille depuis fonts.google.com/specimen/Poppins, dézippe, sélectionne les .ttf manquants, clic droit → "Installer pour tous les utilisateurs".
7

Clés API

Obligatoire (au moins Gemini)

Trois clés API utilisées dans le projet :

  • Gemini (clé-api-gemini.md) — génération d'images IA pour les slides (cover, sections, concepts visuels). La plus utilisée.
  • OpenAI (clé-api-openai.md) — alternative à Gemini pour génération d'images (et analyses entité de marque LLM type notebook V9).
  • DataForSEO (clé-api-dataforseo.md) — data SEO (volumes de recherche, SERP scraping, etc.).
⚠ IMPORTANT

Gemini + OpenAI : remplacer par TES propres clés

Les clés Gemini et OpenAI livrées appartiennent personnellement à l'expéditeur du dossier. Si tu les utilises en production, Yannick va voir une consommation grimper sur ce compte personnel alors que ce n'est pas l'expéditeur qui consomme — ça perturbe le suivi des dépenses agence et risque d'alerter Yannick sur une consommation anormale.

👉 Ce qu'il faut faire dès que tu peux :

  • Soit créer tes propres clés :

Ensuite, éditer clé-api-gemini.md et clé-api-openai.md à la racine du projet et y coller ta clé. C'est la seule étape vraiment manuelle du setup.

DataForSEO (clé-api-dataforseo.md) : c'est la clé de Yannick directement, pas de souci, tu peux l'utiliser et la garder telle quelle. Pas d'action à prendre dessus.
Prompt à coller dans Claude Code (test des clés)
Lis les fichiers clé-api-gemini.md, clé-api-openai.md et clé-api-dataforseo.md à la racine. Pour chaque clé, fais un appel test minimal (ping/auth uniquement, pas de génération volumineuse) pour vérifier qu'elles sont valides et fonctionnelles. Signale-moi celles qui ne marchent pas et propose les liens pour en créer de nouvelles si besoin. RAPPEL CRITIQUE : si je te dis que j'ai mis mes propres clés Gemini ou OpenAI, vérifie spécifiquement qu'elles sont bien personnelles (pas celles de l'expéditeur initial). Pour DataForSEO, c'est normal que la clé soit celle de Yannick, ne pas demander de la remplacer.

8. Bonnes pratiques

Crée un sous-dossier par projet

Pour chaque nouveau client/présentation, crée un dossier dédié à la racine. Exemple : aviva-geo-bilan/, fiducial-pitch/. Mets-y :

  • data-brute/ — fichiers de data source (Excel, CSV, exports)
  • assets/ — images spécifiques au projet (logos client, screenshots)
  • output/ — les .pptx générés
  • scripts/ — le ou les fichiers build_xxx.js
  • README.md — brief du projet, décisions prises

Toujours partir des guides, jamais de l'inspiration

Le workflow Empirik repose sur des règles précises (charte, top-down McKinsey, titres-constats, etc.). Ne demande pas à Claude de "s'inspirer de cette pres" — laisse-le appliquer le système.

Faire valider le plan avant chaque génération

C'est la règle n°1. Une slide générée sans plan validé est une slide à refaire.

9. Dépannage

"Cannot find module 'pptxgenjs'"

Le NODE_PATH n'est pas configuré (Windows) ou les packages npm globaux ne sont pas installés. Re-fais l'étape 4 (§3). Sous Windows, ferme et rouvre le terminal après setx NODE_PATH.

Les titres rendent en Regular au lieu de SemiBold

Les variantes Poppins ne sont pas toutes installées sur ton OS, ou le script utilise fontWeight au lieu de fontFace: "Poppins SemiBold". Vérifie l'étape 6 (§3) + lis PROCESS-ANTI-ERREURS.md erreur #2.

"soffice not found" lors de la conversion PDF

LibreOffice n'est pas installé ou pas dans le PATH. Repasse à l'étape 3 (§3). Sous Windows, appelle directement "C:\Program Files\LibreOffice\program\soffice.exe".

"AF_UNIX is not supported" sous Windows

Tu utilises le wrapper soffice.py au lieu du binaire direct. Appelle soffice.exe directement (chemin complet entre guillemets).

Les images Gemini échouent

Soit la clé API est invalide, soit le quota est dépassé, soit le modèle a changé. Vérifie la clé dans clé-api-gemini.md et le modèle utilisé dans CLAUDE.md §6.4 (par défaut : gemini-3-pro-image-preview).

PyMuPDF échoue à l'import

Installe-le avec pip install PyMuPDF (pas pip install fitz qui est un autre package). Le module Python s'appelle fitz mais le package pip est PyMuPDF.
